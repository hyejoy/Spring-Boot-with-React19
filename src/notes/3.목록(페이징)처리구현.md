# Spring Data JPA: 페이징 & Query Method

## 페이징이 필요한 이유

- DB는 대부분 원격지 → 대량 데이터 전송 부담
- 서버 메모리에서 대량 데이터 관리 부담
- 화면에서도 필요한 만큼만 표시

> SQL의 약 80~90%는 SELECT → 페이징은 필수

---

## Pageable — 페이징 처리

```java
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Sort;

// 1페이지(0부터 시작), 10개씩, tno 기준 내림차순
Pageable pageable = PageRequest.of(0, 10, Sort.by("tno").descending());

Page<Todo> result = todoRepository.findAll(pageable);
```

> ⚠️ 페이지 번호는 **0부터 시작** (1페이지 = 0)

### PageRequest.of() 파라미터

| 순서 | 파라미터 | 설명 |
|------|---------|------|
| 1 | `page` | 페이지 번호 (0부터 시작) |
| 2 | `size` | 페이지당 데이터 수 |
| 3 | `sort` | 정렬 조건 (선택) |

---

## Page\<T\> 반환 타입

`Pageable`을 파라미터로 넘기면 반환 타입은 무조건 `Page<T>`

```java
Page<Todo> result = todoRepository.findAll(pageable);

result.getTotalPages();    // 전체 페이지 수
result.getTotalElements(); // 전체 레코드 수
result.getContent();       // 현재 페이지 데이터 목록
```

### 자동으로 날아가는 쿼리 2개

```sql
-- 1. 데이터 조회
SELECT * FROM tbl_todo ORDER BY tno DESC LIMIT 10;

-- 2. 전체 개수 확인 (다음 페이지 존재 여부 파악)
SELECT COUNT(*) FROM tbl_todo;
```

> `count` 쿼리는 JPA가 **다음 페이지 유무를 파악**하기 위해 자동으로 실행

## Query Method

메서드 이름 자체가 쿼리가 되는 JPA의 강력한 기능.  
`TodoRepository` 인터페이스에 메서드 선언만 하면 자동으로 SQL 생성.

```java
public interface TodoRepository extends JpaRepository<Todo, Long> {

    // title에 키워드가 포함된 데이터를 페이징 조회
    Page<Todo> findByTitleContaining(String keyword, Pageable pageable);
}
```

### 생성되는 SQL

```sql
SELECT * FROM tbl_todo
WHERE title LIKE '%keyword%'
LIMIT 10;
```

### Query Method 네이밍 규칙

| 키워드 | 설명 | 예시 |
|--------|------|------|
| `findBy` | WHERE 조건 시작 | `findByTitle` |
| `Containing` | LIKE '%값%' | `findByTitleContaining` |
| `And` | AND 조건 | `findByTitleAndWriter` |
| `Or` | OR 조건 | `findByTitleOrWriter` |
| `OrderBy` | ORDER BY | `findByTitleOrderByTnoDesc` |

## Query Method의 한계 → QueryDSL

Query Method는 **고정된 조건**만 만들 수 있음.

```
사용자 검색 조건이 동적으로 바뀌는 경우
예) 제목만 검색 / 제목+작성자 검색 / 날짜 범위 검색

→ 조건 조합마다 메서드를 새로 만들어야 함 ❌
```

이런 **동적 쿼리**가 필요할 때 → **QueryDSL** 사용  
(국내 실무에서 가장 많이 쓰이는 방식)

→ 다음 시간에 QueryDSL 세팅 소개 예정

## Dialect (방언) 설정

```yaml
spring:
  jpa:
    database-platform: org.hibernate.dialect.MariaDBDialect
```

- DB마다 SQL 문법이 조금씩 다름 (MySQL, Oracle, PostgreSQL 등)
- Dialect가 각 DB에 맞는 SQL을 자동으로 생성해줌
